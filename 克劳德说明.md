# Claude 风格说明

本文档记录 Claude / Anthropic 营销界面的视觉语言，用作设计参考。正文采用中文；品牌名、字体名、设计 token 和颜色值保留原始写法，便于和实现对齐。

## 总览

Claude.com 的界面气质偏温暖、编辑感和出版感。它不是常见 AI 产品那种冷灰白底，而是以带暖调的奶油色画布 `{colors.canvas}`（`#faf9f5`）作为页面基底。标题使用带出版感的衬线展示字体，例如 `Copernicus` 或 `Tiempos Headline`，字重偏轻，并配合略微收紧的字距；正文和界面标签使用 `StyreneB` 或 `Inter`。

品牌识别主要来自“奶油色 + 珊瑚色”的组合。珊瑚色 `{colors.primary}`（`#cc785c`）是 Anthropic 的核心强调色，用于主按钮、品牌字标强调和整块行动召唤区域。它偏暖、略微低饱和，刻意避开冷蓝、亮青或企业蓝。

页面表面通常在三类模式之间切换：

1. 奶油色画布：默认页面背景。
2. 浅奶油卡片：功能、内容和解释型区域。
3. 深海军黑产品表面：代码编辑器、终端、模型对比、页脚和开发者产品展示。

这种从奶油色到深色产品表面的切换，是 Claude 页面节奏的重要来源。

## 关键特征

- 页面底色使用温暖奶油色 `{colors.canvas}`，文字使用暖黑 `{colors.ink}`。
- 主按钮使用珊瑚色 `{colors.primary}`，但不滥用；单个按钮克制，整块 CTA 区域可以大胆使用。
- 展示标题使用衬线字体，形成文学和编辑感；正文使用人文无衬线字体，保证阅读效率。
- 深色产品卡片用于展示真实产品界面，例如代码、终端、设置面板和模型对比，而不是抽象插画。
- 浅奶油卡片 `{colors.surface-card}` 比页面底色稍深，用于承载解释型内容。
- Anthropic 的放射状标记可作为品牌字标前缀或内容标记。
- 圆角有层级：按钮和输入框约 `8px`，内容卡片约 `12px`，大型展示容器约 `16px`，徽标和胶囊使用全圆角。
- 大区块间距较大，常用 `96px` 作为节奏单位；卡片内边距通常为 `32px`。

## 色彩

### 品牌和强调色

- `{colors.primary}` `#cc785c`：Anthropic 标志性的暖珊瑚色，用于主 CTA、强调卡片和品牌字标。
- `{colors.primary-active}` `#a9583e`：按钮按下或激活时的深色变体。
- `{colors.primary-disabled}` `#e6dfd8`：低饱和的禁用态和边线色。
- `{colors.accent-teal}` `#5db8a6`：少量用于状态点、连接状态或产品界面辅助提示。
- `{colors.accent-amber}` `#e8a55a`：用于分类徽标和局部强调。

### 表面

- `{colors.canvas}` `#faf9f5`：默认页面背景，暖奶油色。
- `{colors.surface-soft}` `#f5f0e8`：柔和分区背景。
- `{colors.surface-card}` `#efe9de`：内容卡片背景。
- `{colors.surface-cream-strong}` `#e8e0d2`：更强调的奶油色表面。
- `{colors.surface-dark}` `#181715`：深色产品展示区、代码卡片和页脚。
- `{colors.surface-dark-elevated}` `#252320`：深色区域内的抬升卡片。
- `{colors.surface-dark-soft}` `#1f1e1b`：深色区域内的代码块底色。
- `{colors.hairline}` `#e6dfd8`：奶油色表面上的细边线。

### 文字

- `{colors.ink}` `#141413`：标题和主要文字。
- `{colors.body-strong}` `#252523`：强调正文。
- `{colors.body}` `#3d3d3a`：默认正文。
- `{colors.muted}` `#6c6a64`：副标题、面包屑和次级文本。
- `{colors.muted-soft}` `#8e8b82`：说明文字、版权和细节文本。
- `{colors.on-primary}` `#ffffff`：珊瑚色按钮上的文字。
- `{colors.on-dark}` `#faf9f5`：深色表面上的主文字。
- `{colors.on-dark-soft}` `#a09d96`：深色表面上的次级文字。

## 字体

展示标题使用 `Copernicus` 或 `Tiempos Headline`，正文和 UI 使用 `StyreneB` 或 `Inter`，代码使用 `JetBrains Mono`。如果没有授权字体，可使用以下替代：

- 展示标题：`Cormorant Garamond`、`EB Garamond` 或 `Tiempos Headline`。
- 正文和 UI：`Inter`，必要时可用 `Söhne`。
- 代码：`JetBrains Mono`。

字体原则：

- 展示标题保持字重 `400`，不要加粗。
- 大标题允许轻微负字距，强化出版感。
- 正文保持 `400`，标签和按钮可用 `500`。
- 不要把展示标题改成普通无衬线，否则会丢失 Claude 的品牌语气。

## 布局

- 基础间距单位为 `4px`。
- 大区块垂直间距约 `96px`。
- 内容最大宽度约 `1200px`，居中排布。
- 英雄区常用左右分栏：左侧标题和按钮，右侧产品展示或插画。
- 功能卡片桌面端常用三列，平板两列，移动端一列。
- 定价卡片桌面端三到四列，移动端单列。

整体留白应当充足，页面更像长篇编辑内容，而不是密集的 SaaS 营销模板。

## 层次和深度

Claude 的深度主要靠颜色块和表面切换，而不是阴影。常见层级：

- 平面层：页面区块、顶部导航和英雄区。
- 细边线层：输入框、卡片边界和局部分隔。
- 奶油色卡片层：功能卡片和内容卡片。
- 深色产品卡片层：代码窗口、终端和模型展示。
- 轻微阴影层：少量悬浮状态，透明度应低。

## 组件参考

### 导航

顶部导航使用奶油色背景，左侧放品牌标记和 `Claude` 字标，中间是产品、方案、用例、价格、研究和公司等菜单，右侧是登录和主 CTA。

### 按钮

- 主按钮：珊瑚色背景、白色文字、`8px` 圆角、高度约 `40px`。
- 次按钮：奶油色背景、细边线、暖黑文字。
- 深色表面次按钮：深色抬升背景、浅色文字，不反转成浅色按钮。
- 文本按钮：无背景，用于导航和正文内链接。
- 圆形图标按钮：用于轮播、分享和更多操作。

### 卡片

- 功能卡片：浅奶油背景、`12px` 圆角、`32px` 内边距。
- 深色产品卡片：深色背景，展示真实产品界面。
- 代码窗口卡片：深色背景、等宽字体、行号和语法高亮。
- 模型对比卡片：浅色背景、细边线，展示 Opus / Sonnet / Haiku 等模型差异。
- 珊瑚色 CTA 卡片：整块珊瑚色背景，白色文字，内部可用浅色按钮。

## 响应式

- 小于 `768px`：导航折叠，英雄区单列，标题缩小，卡片单列。
- `768px` 到 `1024px`：卡片两列，导航略收紧。
- 大于 `1024px`：完整导航、三列功能卡片和多列定价。
- 大于 `1440px`：内容宽度继续保持上限，外侧留白增加。

代码块在移动端应允许横向滚动，不要强行换行破坏代码可读性。

## 应当遵守

- 使用温暖奶油色作为页面基础。
- 展示标题使用衬线字体，并保持轻字重。
- 珊瑚色只用于主行动和高能量 CTA 区域。
- 真实产品界面优先于抽象营销插画。
- 让奶油色、深色产品卡片和珊瑚色 CTA 形成页面节奏。
- 大区块之间保持充足留白。

## 应当避免

- 不要用纯白或冷灰替代奶油色画布。
- 不要把衬线标题加粗到过重。
- 不要使用冷蓝、亮青或大面积紫色作为品牌主色。
- 不要让珊瑚色到处出现。
- 不要连续多个区块使用同一种表面模式。
- 不要给所有卡片增加明显阴影。

## 已知边界

- `Copernicus` 和 `StyreneB` 是授权字体，公开环境通常需要替代字体。
- Anthropic 放射状标记属于品牌资产，不应随意重绘为通用图标。
- 动效细节没有纳入本文档。
- Claude 产品内部界面还有聊天气泡、文件上传、侧边栏等专用组件，本文只描述营销页面表面。
