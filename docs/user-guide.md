# VirtualWebCam 用户使用指南

版本：1.0  
适用对象：项目管理员、现场实施人员、运维人员  
访问地址：`http://<宿主机IP>:5177`，当前示例为 `http://192.168.5.111:5177`

## 1. 系统能做什么

VirtualWebCam 可以把一个网页变成一路虚拟摄像头。创建成功后，你可以得到：

```text
RTSP 地址：rtsp://摄像头IP:8554/流名称
ONVIF 地址：http://摄像头IP/onvif/device_service
go2rtc 页面：http://摄像头IP
```

中控、大屏系统、ONVIF Device Manager 或播放器可以用这些地址接入。

你不需要手动运行 Xvfb、Chrome、FFmpeg、MediaMTX 或 go2rtc。只需要在网页管理后台中创建摄像头即可。

## 2. 基本概念

### 2.1 项目

项目用于区分不同现场、客户或大屏系统。每个项目可以有不同的矩阵屏幕规格。

示例：

```text
总部大厅项目：6 行 x 8 列，共 48 块屏
门店窄屏项目：2 行 x 8 列，共 16 块屏
```

### 2.2 摄像头

一条摄像头记录对应一路网页视频源。摄像头包含：

- 名称。
- 虚拟 IP。
- 网页 URL。
- 流名称。
- 分辨率。
- FPS。
- 运行状态。
- RTSP 和 ONVIF 地址。
- 投放屏幕绑定。

### 2.3 矩阵屏幕

矩阵屏幕用于管理大屏的屏幕编号和绑定关系。它帮助你直观看到：

- 哪一路摄像头投向哪块屏。
- 哪一路摄像头占用了连续几块屏。
- 哪些摄像头还没有绑定屏幕。

### 2.4 围栏区域

如果一个摄像头需要占用连续多块屏，可以在矩阵绑定页面用鼠标画一个矩形围栏。摄像头绑定到围栏后，就表示这一路视频投向这个合并区域。

没有画围栏时，每块屏就是一个独立区域。

## 3. 进入系统

浏览器打开：

```text
http://192.168.5.111:5177
```

页面顶部会显示运行环境状态。

绿色表示：

```text
Docker、macvlan 网络、VirtualWebCam 镜像可用
```

黄色或红色表示环境需要运维处理。常见问题包括：

- Docker 没运行。
- 后端无法访问 Docker socket。
- macvlan 网络不存在。
- `virtualwebcam:latest` 镜像不存在。

## 4. 项目入口

进入系统后先看到项目列表。

每个项目卡片包含：

- 项目名称。
- 矩阵规格，例如 `6x8`。
- 屏幕编号范围，例如 `屏01 - 屏48`。
- 摄像头管理。
- 矩阵绑定。
- 项目设置。

### 4.1 创建项目

在“创建管理项目”区域填写：

- 项目名称。
- 行数。
- 列数。
- 屏幕编号前缀。

点击“新建项目”。

建议命名：

```text
总部大厅大屏
一楼指挥中心
门店窄屏项目
```

### 4.2 进入项目

常用入口：

- “摄像头管理”：创建和维护虚拟摄像头。
- “矩阵绑定”：维护摄像头和屏幕区域的关系。
- “项目设置”：修改项目名称、矩阵规格、导入导出。

## 5. 摄像头管理

摄像头管理页面主要用于查看摄像头详情列表和执行运维操作。

### 5.1 页面信息

顶部统计：

- 总数。
- 运行中。
- 已停止。
- 异常。
- 已绑定。
- 未绑定。

点击统计项可以快速筛选。

列表列信息：

- 名称。
- IP。
- 网页 URL。
- 状态。
- 投放屏幕。
- RTSP。
- ONVIF。
- 操作。

### 5.2 状态含义

```text
运行中：Docker 容器正在运行
已停止：容器不存在或已停止
异常：容器异常退出、Docker 操作失败或状态不可用
```

运行中的摄像头不能重复点击启动。  
已停止的摄像头不能点击停止。  
重启用于让当前配置重新生效。

### 5.3 新增摄像头

点击“新增摄像头”，填写：

- 名称：例如 `web-cam-01`。
- 虚拟 IP：例如 `192.168.5.211`。
- 网页 URL：要显示的大屏网页地址。
- 流名称：例如 `screen01`。
- 宽度：通常 `1280`。
- 高度：通常 `720`。
- FPS：通常 `15`。

点击创建后，系统会：

1. 保存摄像头配置。
2. 创建 Docker 容器。
3. 启动 Xvfb、Chrome、FFmpeg、MediaMTX、go2rtc。
4. 生成 RTSP 和 ONVIF 地址。
5. 在列表中显示状态。

如果创建失败，列表可能显示“异常”，可以打开日志查看原因。

### 5.4 批量生成摄像头

点击“批量生成”。

批量生成只写入摄像头配置，不立即启动容器，适合一次录入几十路摄像头。

需要填写：

- 数量。
- 起始 IP。
- 名称前缀。
- 流名前缀。
- 网页 URL。
- 分辨率。
- FPS。

示例：

```text
数量：8
起始 IP：192.168.5.211
名称前缀：web-cam-
流名前缀：screen
```

生成结果：

```text
web-cam-01 -> 192.168.5.211 -> screen01
web-cam-02 -> 192.168.5.212 -> screen02
...
web-cam-08 -> 192.168.5.218 -> screen08
```

批量生成后，可勾选需要启动的摄像头，点击批量启动。

### 5.5 搜索和筛选

搜索框支持：

- 摄像头名称。
- IP。
- 流名称。
- 网页 URL。
- 屏幕编号。
- RTSP。
- ONVIF。

状态筛选支持：

- 全部状态。
- 运行中。
- 已停止。
- 异常。

### 5.6 单行快捷操作

每行右侧有快捷按钮：

- 启动。
- 停止。
- 重启。
- 配置。

不可执行的按钮会自动禁用，避免误操作。

### 5.7 配置面板

点击“配置”后，会展开该摄像头的操作面板。

常用操作：

- 查看日志。
- 编辑配置。
- 复制为新摄像头。
- 复制 RTSP。
- 复制 ONVIF。
- 打开 go2rtc。
- 删除摄像头。

### 5.8 编辑摄像头

点击“配置” -> “编辑配置”。

可修改：

- 名称。
- IP。
- 网页 URL。
- 流名称。
- 宽度。
- 高度。
- FPS。

如果摄像头正在运行，保存后系统会尝试重建容器，使新配置生效。

修改网页 URL 后，建议重新打开播放器：

```bash
mpv --rtsp-transport=tcp rtsp://192.168.5.211:8554/screen01
```

部分播放器可能停留在旧画面，重新打开即可。

### 5.9 查看日志

点击“配置” -> “查看日志”。

日志可以帮助判断：

- Chrome 是否打开了目标网页。
- FFmpeg 是否正常推流。
- MediaMTX 是否接收到流。
- go2rtc 是否正常启动。
- 容器是否反复重启。

### 5.10 删除摄像头

点击“配置” -> “删除摄像头”。

删除会同时：

- 删除数据库记录。
- 删除对应 Docker 容器。
- 释放该摄像头绑定的屏幕区域。

删除前请确认该摄像头不再被中控使用。

## 6. 矩阵绑定

矩阵绑定页面用于维护“摄像头”和“大屏屏幕”的关系。

### 6.1 页面结构

左侧：

- 未绑定摄像头源。
- 当前围栏信息。
- 已绑定区域。

右侧：

- 屏幕矩阵。
- 单屏区域。
- 合并围栏区域。

### 6.2 单屏绑定

如果摄像头只投向一块屏：

1. 在左侧找到未绑定摄像头。
2. 拖拽到目标屏幕。
3. 松开鼠标。

绑定成功后：

- 该摄像头从“未绑定摄像头源”列表中移除。
- 屏幕格子中显示摄像头信息。
- 摄像头列表中的“投放屏幕”会显示对应屏幕。

### 6.3 合并区域绑定

如果一个摄像头需要占用连续几块屏：

1. 在矩阵上按住鼠标。
2. 拖出一个矩形区域。
3. 松开鼠标，形成围栏。
4. 把未绑定摄像头拖入这个围栏。

示例：

```text
屏01、屏02、屏03
屏09、屏10、屏11
```

这是一个 `3列 x 2行` 的合并区域。

绑定后会显示：

```text
摄像头名称
屏01 - 屏11 · 3列x2行
IP
流名称
状态
```

围栏保持透明，可以看到被圈住的屏幕编号。

### 6.4 解除绑定

在已绑定区域或矩阵围栏上点击解除按钮。

解除后：

- 摄像头回到未绑定列表。
- 屏幕或围栏恢复为空闲。
- 可以重新拖拽绑定。

### 6.5 绑定限制

- 一个屏幕同一时间只能绑定一路摄像头。
- 摄像头绑定后不会继续出现在未绑定列表中。
- 多屏合并必须是连续矩形。
- 不支持一个摄像头绑定多个分散区域。
- 不支持两个摄像头共享同一块屏。

这些限制是为了避免现场投屏关系混乱。

## 7. 项目设置

项目设置用于调整项目级信息：

- 项目名称。
- 行数。
- 列数。
- 屏幕编号前缀。
- 导出项目配置。
- 导入项目配置。

### 7.1 修改矩阵规格

修改行列数前需要注意：

- 如果缩小矩阵，超出范围的屏幕绑定可能需要重新整理。
- 建议先解除不确定的绑定，再调整规格。
- 调整后到矩阵绑定页面检查显示是否正确。

### 7.2 导出项目

导出用于备份或迁移项目。

导出内容包含：

- 项目信息。
- 矩阵规格。
- 摄像头配置。
- 屏幕绑定。
- RTSP/ONVIF 地址。

### 7.3 导入项目

导入时如果发现 IP 冲突，系统会尝试在同网段寻找可用 IP，并显示重映射结果。

导入后建议：

1. 检查摄像头 IP。
2. 检查矩阵绑定。
3. 分批启动摄像头。
4. 逐路验证 RTSP/ONVIF。

## 8. 操作审计

操作审计记录关键变更：

- 创建项目。
- 修改项目。
- 导入项目。
- 修改矩阵。
- 创建摄像头。
- 编辑摄像头。
- 绑定屏幕。
- 启动、停止、重启。
- 删除摄像头。

审计用于排查“谁改了配置”和“什么时候改了配置”。

## 9. 接入中控

### 9.1 ONVIF 手动添加

中控支持 ONVIF 手动添加时填写：

```text
协议：ONVIF
IP：192.168.5.211
端口：80
用户名：空
密码：空
```

如果要求完整地址：

```text
http://192.168.5.211/onvif/device_service
```

### 9.2 RTSP 直连

中控支持 RTSP 时填写：

```text
rtsp://192.168.5.211:8554/screen01
```

### 9.3 ODM 测试

ONVIF Device Manager 手动添加：

```text
http://192.168.5.211/onvif/device_service
```

成功后应能看到：

```text
Profile：screen01
Stream URI：rtsp://192.168.5.211:8554/screen01
Live Video：网页画面
```

## 10. 播放器测试

推荐：

```bash
mpv --rtsp-transport=tcp rtsp://192.168.5.211:8554/screen01
```

也可以使用：

```bash
ffplay -rtsp_transport tcp rtsp://192.168.5.211:8554/screen01
```

如果修改了网页 URL 或重启了容器，播放器可能不会自动刷新。关闭播放器重新打开即可。

## 11. 推荐命名和 IP 规划

### 11.1 IP 规划

当前地址池：

```text
192.168.5.208/28
```

建议：

```text
192.168.5.210  宿主机 macvlan-host 辅助接口
192.168.5.211  摄像头 01
192.168.5.212  摄像头 02
192.168.5.213  摄像头 03
...
```

不要把摄像头 IP 设置成网关、宿主机 IP 或已被其他设备占用的 IP。

### 11.2 摄像头命名

建议名称体现用途：

```text
大厅信息屏
生产看板
会议室日程
西南片区数据屏
```

### 11.3 流名称

建议使用英文、数字和短横线：

```text
screen01
screen02
lobby-info
meeting-schedule
```

不要使用中文、空格或特殊符号。

## 12. 常见问题

### 12.1 为什么页面显示运行中，但画面是网页错误页？

说明 RTSP 管道正常，但 Chrome 打不开目标网页。请检查网页地址、网络、DNS、证书和登录权限。

### 12.2 修改网页 URL 后为什么播放器还是旧画面？

摄像头容器重启后，部分播放器会停留最后一帧。关闭播放器后重新打开即可。

推荐用 mpv 验收，因为源断开时它通常会退出。

### 12.3 为什么宿主机本机打不开摄像头 IP？

这是 macvlan 的常见限制。运维需要配置 `macvlan-host` 辅助接口。其他同网段机器通常可以直接访问。

### 12.4 为什么 ONVIF 自动发现不到？

自动发现依赖 UDP 3702 和网络广播策略。当前主要支持手动添加。只要手动添加成功并能看到 Live Video，就说明可接入。

### 12.5 为什么一个屏幕不能绑定多个摄像头？

为了避免现场关系混乱，一个屏幕同一时间只能属于一个摄像头区域。如果需要一个摄像头占用多块屏，请使用矩形围栏合并区域。

### 12.6 批量生成为什么不直接启动？

几十路摄像头同时启动会带来较大 CPU、内存和网络压力。批量生成先写配置，现场可以分批启动，便于观察资源和故障。

### 12.7 换了网络环境或网段后要怎么处理？

如果现场网段从 `192.168.5.xxx` 变成 `192.168.9.xxx`，需要运维重新创建 Docker macvlan 网络、重新配置宿主机 `macvlan-host` 辅助接口，并把管理后台里的摄像头虚拟 IP 改到新网段。

摄像头镜像不用重新构建。详细命令见《部署运维文档》的“更换网络环境 / 网段迁移”章节。

## 13. 日常操作建议

- 创建摄像头后先用 go2rtc Web 或 mpv 验证。
- 再用 ODM 手动添加验证 ONVIF。
- 最后接入中控。
- 批量摄像头分批启动。
- 大屏静态页面优先使用 `1280x720@15` 或更低 FPS。
- 修改 URL 后重新打开播放器确认。
- 删除摄像头前确认中控侧已经不再使用。
